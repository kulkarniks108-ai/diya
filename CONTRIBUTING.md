# Contributing Guidelines

Thank you for your interest in contributing to this project.

This repository is maintained under a proprietary license.
Contributions, collaborations, and participation are subject
to review and approval by the repository owner.

## Contribution Process

Before contributing, please:

1. Open an issue describing:
   - the proposed change
   - motivation
   - implementation approach

2. Wait for approval before beginning work.

3. Use a dedicated branch for all changes.

4. Submit contributions through pull requests only.

## Code Standards

Contributors are expected to:

- Write clean and maintainable code
- Follow existing project architecture
- Use meaningful commit messages
- Avoid unnecessary dependencies
- Document significant changes

## Restrictions

By contributing to this repository, you agree that:

- Contributions may be modified or rejected
- Ownership and licensing remain with the repository owner
- Unauthorized redistribution or reuse is prohibited
- Contributions must not violate third-party rights

## Security

Do not commit:
- API keys
- credentials
- secrets
- production tokens

## Conduct

Professional and respectful collaboration is expected
at all times.
